## Revision Strategies

### Early Revision

Catch errors early:

```
T1: Initial understanding

T4: Corrected approach

**Benefit:** Prevents building on false premises.

---

### Late Revision

Discover issues near completion:

```
T1-T5: Build solution
T6: Verify... wait, doesn't handle edge case X
T7 [REVISE T5]: Adjusted solution for edge case
T8: Re-verify - now complete
```

**Benefit:** Ensures thorough validation.

---

### Multiple Revisions

Don't hesitate to revise repeatedly:

```
T1: Understanding
T2: Approach A
T3 [REVISE T2]: Actually, approach B
T4: Details of B
T5 [REVISE T3]: Wait, hybrid A+B is better
T6: Final hybrid approach
```

**Benefit:** Shows genuine problem-solving process.

- Every branch has a purpose

- Better to revise than persist with errors
- Better to explore than commit too early

## Parameter Decision Tree

### Starting a Sequence

```
1. Set thought_number = 1
2. Estimate total_thoughts (usually 5-7)
3. Write your first thought
4. Set next_thought_needed = true
```

### Continuing

```
1. Increment thought_number
2. Adjust total_thoughts if needed
3. Write next thought
4. Set next_thought_needed = true (unless done)
```

### Revising

```
1. Set is_revision = true
2. Set revises_thought = <number>
3. Write revised thinking
4. Adjust total_thoughts if needed
```

### Branching

```
1. Set branch_from_thought = <number>
2. Set branch_id = "<name>"
3. Write alternative approach
4. Continue branch or merge back
```
